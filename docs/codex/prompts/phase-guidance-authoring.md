# Codex task: write or revise phase guidance

## Goal

Write or revise a phase guidance document for one bounded development phase.

## Required structure

The guidance must include:

1. phase purpose
2. allowed work
3. forbidden work
4. input artifacts
5. output artifacts
6. data structures, if applicable
7. status labels and precedence, if applicable
8. claim-boundary rules
9. non-inference rules
10. deterministic-output rules
11. required tests
12. required validation commands
13. final report expectations

## Required boundary

Do not expand the scientific claim level of the repository.

Do not introduce language implying:

- theorem-level deployment verification
- runtime selector-regime proof
- bridge-qualified V-information proxy certification without fresh matching bridge evidence
- CandidatePool as a core paper artifact
- gamma_hat legacy/non-headline alias as a submodularity ratio
- block_ratio_lcb_star as a paper-grade degree-adaptive estimator

## Review requirement

After drafting, the guidance must be reviewed using:

- `docs/codex/guidance-document-review.md`
- `docs/codex/prompts/guidance-document-review.md`
