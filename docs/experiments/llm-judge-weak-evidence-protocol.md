# LLM Judge Weak Evidence Protocol

This protocol defines a static harness for weak, model-adjudicated candidate evidence. It does not establish human/external gold labels, calibrated judge behavior, measurement validation, paper-grade validation, selector superiority, Route 5 unlock, or Route 8 unlock.

## Claim Boundary

- Claim status: `operational_utility_only/no_claim_upgrade`
- Allowed claim level: `model_adjudicated_weak_evidence`
- Route 5 locked: true
- Route 8 locked: true
- Raw provider bodies stored: false
- Live API calls performed by the harness: false

Judge outputs are candidate diagnostics only. Any downstream report must describe them as weak/model-adjudicated evidence and must suppress or downgrade the claim when stability checks fail.

## Labels

Allowed labels are:

- `support`
- `insufficient`
- `contradict`
- `uncertain`
- `parse_failed`

The parser maps unrecognized labels to `parse_failed` and tracks the failure in the record flags.

## Required Controls

- Blinded pairwise comparison
- Order-swapped duplication
- Duplicate judging with a minimum of two judgments
- Rubric paraphrase IDs with a minimum of two variants
- Temperature fixed at `0`
- Length-aware reporting via input and output token counts
- Parse failure tracking
- `raw_response_stored=false`

## Stability Gate

The claim gate reports `weak_evidence_candidate_ready` only when all required stability checks pass:

- Parse failure rate <= `0.15`
- Order-swap agreement >= `0.80`
- Duplicate agreement >= `0.80`
- Rubric paraphrase agreement >= `0.75`

If any threshold fails, the gate returns `downgraded_to_ambiguous`, suppresses `model_adjudicated_weak_evidence`, and reports `ambiguous_suppressed`.

## Prompt Assets

- `prompts/judge/weak_evidence_v1.md`
- `prompts/judge/weak_evidence_v1_order_swapped.md`

Prompt hashes are generated by `cps.judge.judge_manifest.prompt_hashes()` and are included in each deterministic judge run manifest.
